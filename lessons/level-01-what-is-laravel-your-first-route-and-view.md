## TL;DR

Laravel is a PHP web framework — opinionated, elegant, batteries included. It gives you routing, ORM, templating, authentication, queueing, and more out of the box. The most popular PHP framework in the world.

## Real-world analogies

- Laravel is a fully-equipped workshop. PHP gives you tools; Laravel gives you a workshop with the right tool for every job, organized and ready.
- If PHP is cooking with raw ingredients, Laravel is meal-kit delivery: every recipe (route), every ingredient (model), every technique (validation) is prepared for you.

## Key concepts

### `MVC pattern`

Model (data + business logic), View (HTML output), Controller (handles requests). Laravel is built on this.

### `Routing`

URLs map to controllers. `/users` → `UserController@index`. Laravel's `routes/web.php` defines these.

### `Artisan CLI`

Laravel's command-line tool. `php artisan make:controller`, `php artisan migrate`, `php artisan serve`. Your best friend.

### `Composer`

PHP's package manager. Laravel itself is installed via Composer. So are most Laravel packages.

## Code with comments

Every line has a comment. Read it slowly.

```
<?php
// === Installing Laravel ===
// composer create-project laravel/laravel my-app
// cd my-app
// php artisan serve

// === routes/web.php — your first route ===
use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('welcome');  // Returns the welcome.blade.php view
});

Route::get('/hello/{name}', function ($name) {
    return "Hello, {$name}!";
});

// Route with a controller
Route::get('/users', [UserController::class, 'index']);

// === resources/views/welcome.blade.php ===
// <!DOCTYPE html>
// <html>
//   <body>
//     <h1>Welcome to Laravel</h1>
//     <p>{{ date('Y-m-d H:i:s') }}</p>
//   </body>
// </html>

// === Blade templating (Laravel's templating engine) ===
// {{ $variable }}    — escaped output (safe from XSS)
// {!! $rawHtml !!}   — unescaped (dangerous with user input)
// @if ($condition) ... @endif
// @foreach ($items as $item) ... @endforeach
// @csrf              — CSRF protection token
// {{ route('users.show', $user->id) }}  — Generate URLs

// === A simple controller ===
// php artisan make:controller HelloController

namespace App\Http\Controllers;

class HelloController extends Controller
{
    public function greet(string $name)
    {
        return view('hello', ['name' => $name]);
    }
}

// === resources/views/hello.blade.php ===
// <h1>Hello, {{ $name }}!</h1>
// <p>Today is {{ now()->format('l, F j, Y') }}</p>

// === Passing data to a view ===
public function show()
{
    $data = [
        'title' => 'My Page',
        'users' => ['Aby', 'Arsha', 'Foo'],
        'count' => 42,
    ];
    return view('page', $data);
}

// === Your first Artisan commands ===
// php artisan list                    — List all commands
// php artisan make:controller Foo     — Create a controller
// php artisan make:model Bar          — Create a model
// php artisan migrate                 — Run database migrations
// php artisan serve                   — Start dev server
// php artisan tinker                  — Interactive PHP REPL with Laravel loaded
```

## Try it yourself

Install Laravel: `composer create-project laravel/laravel hello-app`. Create a route `/hello/{name}` that returns a Blade view saying 'Hello, {name}!'. Run `php artisan serve` and visit it in the browser.

## Common pitfalls

- ⚠️ Forgetting to run `php artisan key:generate` after a fresh install. Sessions and encryption won't work without it.
- ⚠️ Putting business logic in routes/web.php. Use controllers instead — keeps things organized.
- ⚠️ Using `{!! !!}` (unescaped) for user input. Use `{{ }}` (escaped) to prevent XSS attacks.
